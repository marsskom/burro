# Burro v0.1.0 — Initial Release

Burro v0.1.0 is the first functional release of a modular HTTP proxy and traffic inspection runtime designed around a plugin-first architecture.

This version establishes the core execution model, plugin system, TLS interception pipeline, and traffic observability foundation.

---

## What Burro is in v0.1.0

Burro is not just a proxy — it is a runtime for inspecting and transforming network traffic via plugins.

In this release, Burro provides:

- A working HTTP/HTTPS proxy with interception support
- A plugin execution system for request/response lifecycle control
- A configurable runtime environment (runtime/)
- Traffic export and observability features
- Optional gRPC event streaming interface
- TLS interception via custom CA certificates
- HTTP streaming and WebSocket support
- Lua embedded scripts system

---

## Core Capabilities

HTTP Proxy with Interception

Burro can:

- Intercept HTTP and HTTPS traffic
- Inspect full request/response lifecycle
- Modify traffic via plugins before it is sent or returned
- Save data into sqlite db

Supports:

- Transparent request processing pipeline
- Full request/response snapshots (depending on hook stage)
- Streaming-safe handling for large responses (limited snapshots where needed)

---

## Plugin-Based Architecture (Core Feature)

Burro v0.1.0 introduces a strict plugin system where behavior is fully externalized.

Plugins:

- are loaded from `plugins/` (golang) or `%workdir%/plugins/luaplugin/scripts` (lua)
- are configured via `%workdir%/config.yml`
- operate through well-defined lifecycle hooks
- are isolated from core internals

Plugin model:

- each plugin has:
  - local config (`%workdir%/plugins/%name%/config.yml`)
  - artifact directory (`%workdir%/artifacts/%name%/`)
  - data directory (`%workdir%/plugins/%name%/`)

Plugin rules:

- no direct access to core internals
- no bypassing plugin API
- only lifecycle hooks + event-driven execution
- isolated configuration and storage

---

## Plugin Hooks (v0.1.0 Core Set)

HTTP lifecycle

- `OnConnect`
- `OnBeforeRequestSend`
- `OnAfterRequestSend`
- `OnBeforeResponseSend`
- `OnAfterResponseSend`
- `OnError`
- `OnClose`

WebSocket lifecycle

- `OnWSOpen`
- `OnWSMessage`
- `OnWSClose`

Special hook for plugins need to create some files

- `Flush`

These hooks define the entire execution surface of Burro v0.1.0 plugins.

## Built-in Core Plugins

Burro ships with a minimal but functional plugin set:

- **Logger** — main logging
- **Policy** — request filtering / blocking logic
- **HAR Export** — exports traffic into HAR-like files
- **Lua** — runtime scripting engine (major extensibility feature)

---

## Lua Plugin System (Experimental but Functional)

One of the most powerful parts of v0.1.0.

Burro embeds a Lua runtime with access to:

**Core objects**

- `ctx` — request/response context snapshot
- `mut` — mutation API for request/response
- `log` — structured logging
- `bus` — internal event bus
- `kv` — key-value storage
- `artifacts` — persistent file storage
- `data_store` — read-only datasets
- `time`, `json`, `base64` packages

### Script lifecycle entrypoints

Lua scripts map directly to events:

- `connect.lua`
- `before_request.lua`
- `after_request.lua`
- `before_response.lua`
- `after_response.lua`
- `error.lua`
- `flush.lua`
- `close.lua`
- `ws_open.lua`
- `ws_msg.lua`
- `ws_close.lua`

### Mutation capabilities

Lua scripts can:

- modify request:
  - headers, body, method, URL, cookies
- modify response:
  - status, headers, body
- terminate processing early via:
  - `mut.ctx.set_finish()`

## TLS Interception (CA-based)

Burro supports HTTPS interception via custom CA:

Features

- generate CA certificate (burro cert init)
- install CA into system trust store (MacOS)
- generate host certificates dynamically
- intercept HTTPS traffic via MITM model

---

## Traffic Export

Burro supports session-based persistence:

- workspace mode (`-w <name>`)
- SQLite-based session storage
- optional HAR export via plugin system
- artifact-based file outputs in `%workdir%/artifacts/`

---

## Zero-Configuration Mode

Burro can run without runtime directory and configuration:

```sh
burro proxy -z localhost:8080
```

In this mode:

- logger plugin only
- no persistence
- no plugin system initialization (except core logging)
- useful for quick inspection/testing

---

## gRPC Observability API (Experimental)

v0.1.0 introduces optional gRPC stream interface:

Capabilities:

- live HTTP event streaming
- request/response lifecycle observation
- WebSocket event streaming
- external integrations (dashboards, SIEM, analyzers)

Use cases:

- real-time traffic monitoring
- external security tools
- debugging and observability pipelines

---

## File Server Utility

Burro includes a simple static file server:

```sh
burro serve localhost:8888 ./runtime
```

Optional TLS support via generated certificates.

## Architecture Summary (v0.1.0)

- Core is minimal and stable
- Plugins define behavior
- Traffic flows through hook pipeline
- Lua adds runtime scripting layer
- gRPC exposes observability stream
- Artifacts + workspace enable persistence

## Status of v0.1.0

This is an **initial stable architecture release**, not a production-hardened security tool.

Known characteristics:

- API may evolve in future releases
- plugin interfaces may be extended
- Lua runtime is experimental but functional
- gRPC schema may change

- extensible traffic manipulation engine
